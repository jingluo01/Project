## 二、 普通用户功能指南 (学生/教职工)

普通用户登录后，主要使用以下核心业务流：

### 2.1 实时车位查询与预约
1. 点击左侧导航栏的 **[停车场地图]**。
2. 页面会实时展示当前各个区域的剩余车位情况（通过 WebSocket 技术实现毫秒级更新）。
3. 选择空闲车位，点击 **[预约]** 按钮。
4. 系统会要求录入车牌号。确认后，成功锁定该车位（车位状态变更为“已预约”）。
   *(注：预约后有一定的免费保留时间，详见系统配置。超时未入场将自动取消并可能扣除信用分)*。

### 2.2 模拟入场与出场 (核心业务流)
由于没有真实物理道闸，系统内提供了模拟入出场功能：
1. **入场**: 预约成功后，点击界面的“模拟入场”或扫描生成的二维码，系统记录入场时间，开始计费。
2. **出场**: 点击“模拟出场”或扫描出场二维码，系统根据您的入场时间和用户角色（学生9折/教职工8折）自动计算停车费。

### 2.3 自动扣费与支付宝沙箱支付
出场产生费用后：
1. **余额充足**: 如果您的账户余额/信用分充足，系统将开启“无感支付”，自动从钱包扣款，直接放行，极大提升通行效率。
2. **余额不足 (欠费状态)**: 订单进入“待支付”状态。
3. **支付宝充值**: 前往 **[个人中心]** -> **[钱包/充值]**，系统集成了 **支付宝沙箱支付体系**，扫码付款后，由于系统对接了支付宝异步回调(Webhook)，钱款将自动到达系统账户。重新支付欠款订单即可。

### 2.4 个人信用中心
前往 **[个人中心]** 可以查看个人信用分。
- 按时履约、正常缴费将维持 100 分完美信用。
- 若预约后长时间违约未入场，或者长期拖欠停车费，系统将扣除您的信用分。
- **惩罚机制**: 当信用分低于及格线（70分）时，将被系统限制预约功能。

## 三、 管理员功能指南 (Admin)

管理员登录后，将进入后台大盘，负责整个停车场的运营与监控。

### 3.1 核心数据大盘 (Dashboard)
登录后默认进入 **[控制台 Dashboard]**。
在这里可以直观地看到：
- 今日总收入、实时入场车辆数。
- 车位整体利用率（饼状图/柱状图）。
- 实时滚动的出入场日志。

### 3.2 停车场地与费率管理
前往 **[车位管理]** / **[区域管理]** 模块：
1. **动态费率控制**: 管理员可以随时调整 A区、B区的停车费率和免费时长。由于系统底层使用了 **高并发 Redis 缓存技术**，修改费率后系统会自动清空缓存并立即生效。
2. **车位管控**: 若某个车位正在维修，管理员可强制将其状态修改为“维护中”，前台用户将无法预约该车位。

### 3.3 订单与财务审计
前往 **[订单管理]**：
- 这里记录了每一笔停车订单的详细生命周期（预约时间 -> 入场时间 -> 出场时间 -> 支付时间）。
- 管理员可追踪是否有“逃单”（状态为待支付的滞留订单），并可进行人工干预或强制结算。

### 3.4 师生白名单对接 (特色功能)
前往 **[用户管理]** 模块：
- 系统不仅有普通用户表，底层还打通了 **学校官方教务数据库 (school_official)**。
- 只有真实学号/工号才能完成注册并享有折扣权益，严防外部社会车辆恶意注册占用资源。管理员可在此审核和管理所有注册用户信息。

*(注：系统内置的用于注册测试的【教务系统白名单列表】请参见本文第一章 1.3 节)*
